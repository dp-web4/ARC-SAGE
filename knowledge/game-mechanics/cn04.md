# cn04 — Jigsaw Connector: Align Sprite Connectors by Moving and Rotating Pieces

## Toolset
Available actions: [ACTION1=UP, ACTION2=DOWN, ACTION3=LEFT, ACTION4=RIGHT, ACTION5=ROTATE, ACTION6=CLICK].
Click selects/deselects sprites. Arrows move the selected sprite. Rotate turns it 90 degrees CW. This is a spatial puzzle game about fitting connectors together.

## Objects
- **~23 uniquely-shaped sprites** (various names): Each is a multi-colored pixel shape with **connector pins** marked by color `8` (grey). Colors include `10` (green), `11` (cyan), `12` (blue), `13` (magenta), `14` (yellow), `15` (white), `6` (pink), `7` (orange). All have `tags=["sys_click"]` -- they are clickable and interactive.
- **L-shaped bracket sprites** (`ezhdijgxgn`, `kodeocvgpm`): Color `6`/`7`, NO `sys_click` tag -- these are static/decorative (not clickable).
- **Step counter UI** (`qdcvayjdkm`): Renders a progress bar on row 0 of the 64x64 frame, color `4` (filled) vs `0` (empty), 32px wide centered.
- **Camera**: Background and letterbox colors change per level (from `BackgroundColour` in level data).

## Interaction Model
- **CLICK (ACTION6)**: Uses `display_to_grid()` to convert pixel coords to grid coords. Finds sprite at that grid position. If clicking the already-selected sprite, it deselects. If clicking a different sprite, it selects it. Selection changes sprite visuals:
  - **Selected sprite** (non-level-5): all non-transparent, non-grey, non-green pixels turn to color `0` (black). On level 5+: reveals original colors.
  - **Deselected sprite** (level 5+): all visible pixels turn to color `4` (hidden/uniform).
- **Movement (ACTION1-4)**: Moves the selected sprite by 1 cell in the given direction. Bounded by grid_size (20x20). After moving, recalculates connector overlaps and checks win.
- **ROTATE (ACTION5)**: Rotates the selected sprite 90 degrees CW. Then recalculates overlaps and checks win.
- **Connector matching system** (`gjhtwbvrel`): After any move/rotate, the game finds all grey pixels (`8`) that overlap between two sprites at the same world coordinate. Overlapping grey pixels turn green (`3`) on both sprites. The rotation-to-local-pixel mapping handles all 4 rotation states.

## Win Condition
Method `exlcvhdjsf()`: iterates every sprite. For each sprite, checks its original pixel data. Any grey pixel (`8`) that overlaps with another sprite's grey pixel has been converted to green (`3`). **Win = no grey pixels (`8`) remain on ANY sprite.** Every connector pin must be matched with exactly one other sprite's connector pin at the same world position.

## Level Progression
**5 levels**, all on 20x20 grids. Changes between levels:
- **Level 1**: 2 sprites, background color 10
- **Level 2**: 3 sprites, background color 12
- **Level 3**: 3 sprites, background color 15
- **Level 4**: 3 sprites with **rotations** (90, 270), background color 12
- **Level 5**: 4 sprites with rotations, background color 9. **Major mechanic change**: all non-selected sprites are hidden (all pixels set to color `4`). Only the selected sprite shows its true colors. This is the "blind" level.

Key data per level: `BackgroundColour` (int), list of sprites with positions and optional rotations.

Step limit: 150 actions across all levels.

## No-Source Discovery Strategy
1. **First click**: Click on a visible shape. Observe it changes color (gets highlighted/selected).
2. **Observe**: Note the grey dots on sprites -- these are connector pins. After selecting, try arrow keys to move the shape.
3. **Hypothesis**: Grey dots turn green when two sprites' grey dots overlap at the same position. The goal is to connect all grey pins.
4. **Test**: Move a sprite so its grey pin overlaps another sprite's grey pin. Confirm it turns green.
5. **When ready**: Once all grey pins are green, the level auto-advances. On level 5, note that unselected sprites become invisible -- you must remember positions.

## Key Patterns
- Grid size is always 20x20.
- Color `8` = unmatched connector pin, color `3` = matched connector pin.
- Level 5 hides all unselected sprites (color `4`) -- must track positions mentally.
- Sprites are initially selected to the one closest to origin (min x^2 + y^2).
- Rotation affects connector pin local-to-world mapping with standard 90/180/270 transforms.
- 150-action hard limit; exceeding it = lose.
- Coordinate mapping: `display_to_grid()` from camera handles the pixel-to-grid conversion.
