# sc25 -- Spell-casting movement puzzle with pattern matching and teleportation

## Toolset
Available actions: [1, 2, 3, 4, 6] (UP, DOWN, LEFT, RIGHT, CLICK).
This is a grid-based movement puzzle with a spell-casting subsystem. The player moves a character, clicks on a 3x3 spell grid to toggle spell slots, and triggers spell effects (teleport, fireball, size change) to reach an exit.

## Spell System
Three spells defined by 3x3 boolean patterns in `spell_legends`:
- **`jzukcpajs`**: Pattern `[[T,T,F],[F,T,F],[F,F,F]]` -- top-left L shape
- **`fpokrvgln`**: Pattern `[[F,T,F],[T,F,T],[F,T,F]]` -- diamond/cross
- **`aprnrzeyj`**: Pattern `[[F,T,F],[F,T,F],[F,T,F]]` -- vertical line

Spell casting: Toggle cells in the 3x3 spell grid via CLICK. When the active pattern matches a spell that is in the level's `wdsxxkugj` (available spells list), it auto-casts. Spell effects include teleportation (`qnydzqpivk`), fireball (`aevhgfjqhy`), and size change (`idmgsqivdp`).

## Interaction Model
- **ACTION1-4 (Movement)**: Move player by `ohhczuvpmt` pixels (2 if scale=1, 4 if scale=2). Uses `try_move()` with collision detection. Player rotates to face direction.
  - Scale 2 with step 4: also attempts a half-step follow-up move.
  - Collecting energy packs reduces action count.
  - Colliding with `pcohqadae` (exit) triggers exit animation -> level complete.
- **ACTION6 (CLICK)**: `display_to_grid` maps click coordinates.
  - Clicking a **spell slot** (`bbyayvbjq` / `jqwvpaczd`): toggles that cell, increments `actions_taken`, checks for spell pattern match.
  - Clicking a **spell legend** (`lppppobad-*`): highlights that spell or demos its pattern.
  - Clicking elsewhere: triggers flash animation (invalid click feedback).

Multiple animation systems block input during playback:
- `entering_exit`: 2-frame walk-into-exit
- `spell_slots_flash_animation`: 12-frame flash for invalid clicks
- `spell_pattern_demo_animation`: highlights spell pattern slots sequentially
- `auto_cast_animation`: 8-frame spell casting sequence
- `teleport_animation`: 4-frame teleport with color flash
- `size_change_animation`: 8-frame scaling
- `fireball_animation`: 6-frame fireball effect

## Win Condition
The player wins by colliding with the `pcohqadae` (exit door) sprite during movement. This triggers `qddsvybuqj()` which plays the exit animation, then calls `next_level()` via `ylcthgnbuv()`.

Loss: `actions_taken > sykpecmoq` (action budget exceeded). Budget is per-level, set in level data.

## Level Progression
6 levels. Each level specifies:
- `sykpecmoq`: maximum action budget (None = unlimited)
- `wdsxxkugj`: list of available spell names for that level
- Different arrangements of walls, exits, teleport targets, doors, energy packs
- First 3 levels highlight a spell for tutorial purposes

## No-Source Discovery Strategy
1. Move in each direction to confirm movement and observe step size.
2. Look at bottom of screen: 3x3 grid of cells. Click one cell to see it toggle.
3. Click multiple cells to form patterns; observe which triggers a spell.
4. Note spell legend icons on screen; click them to see pattern demos.
5. After casting a spell, observe the effect (teleport, size change, fireball).
6. Move toward the exit sprite (identifiable by distinct shape).
7. Pick up energy packs (they disappear on contact) to regain actions.
8-10. Map which spells are available per level and plan route + spell usage.

## Key Patterns
- Movement step size depends on player scale: small (scale=1) moves 2px, large (scale=2) moves 4px.
- Spells are the core puzzle mechanic; each level only allows specific spells.
- Teleport cycles through destination list; repeated casts cycle to next target.
- Size change can be blocked if target scale causes collision.
- Energy packs are critical for action economy on tight-budget levels.
- The 3x3 spell grid is located at pixel coordinates (24-34, 49-59) in game space.
- Invalid clicks (not on spell slots or legends) consume animation frames but do not count as actions.
- First input on level 1 triggers a spell pattern demo animation.
