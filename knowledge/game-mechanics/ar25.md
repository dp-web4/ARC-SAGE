# ar25 -- Mirror-reflection puzzle with movable shapes and axis lines

## Toolset
Available actions: [1, 2, 3, 4, 5, 6, 7] (UP, DOWN, LEFT, RIGHT, CYCLE_SELECT, CLICK, UNDO).
This is a spatial puzzle where you position colored shapes and reflection axes on an 8x8 grid so that reflected images cover target dots.

## Objects
Grid size: 8x8 per level. Single sprite ("sprite-1", color 9, 1x1) used as level placeholder; actual objects are loaded dynamically.

| Tag | Role | Description |
|-----|------|-------------|
| `gljpmsnsnx` | **Movable shapes** | Colored polyomino sprites (various colors: 5=gray, 8=azure, 9=maroon, 11=teal, 12=blue, 14=magenta, 15=white). Moved with arrow keys when selected. |
| `edyhkfhkcf` | **Reflection axes** | Lines that act as mirrors (color 10=orange). Two subtypes below. |
| `zxikvwjsyl` | **Vertical axis** | A tall 1-wide column of orange (10). Reflects horizontally: `new_x = 2*axis_x - old_x`. Can only move left/right. |
| `ezdsyuixsn` | **Horizontal axis** | A wide 1-high row of orange (10). Reflects vertically: `new_y = 2*axis_y - old_y`. Can only move up/down. |
| `vrfjzqaker` | **Target dots** | Single teal (11) pixels at specific positions. Must be covered by reflections. Layer -4. |
| `pwbzvhvyzx` | **Fixed/immovable** | Shapes tagged with this cannot be moved or selected. |
| `xnpsicjqxc` | **Horizontal-only reflect** | Shapes that only reflect across the vertical axis. |
| `fmxjsieygg` | **Vertical-only reflect** | Shapes that only reflect across the horizontal axis. |
| `dmfgetmeus` | **Auto-rotate on axis move** | Shapes that rotate 90 degrees when their associated axis crosses them. |
| `sys_click` | **Clickable** | Sprites selectable by clicking. |
| `flrtaztmgm` | **Reflection overlay** | Full-grid overlays (layer 0, -1, -2) that display computed reflections. |
| `sllpppnezc` | **Selection cursor** | Black L-shaped outline (color 0) on layer 5, positioned at selected sprite. |

Color palette: 0 (black/cursor), 5 (gray), 8 (azure), 9 (maroon), 10 (orange axes), 11 (teal targets), 12 (blue), 14 (magenta), 15 (white).

Step counter UI rendered by `hpnnoufcuc` class. Border overlays are also drawn by `gdycxeziaj`.

## Interaction Model
- **ACTION1 (UP)**: Move selected sprite up by 1px. Vertical axis: no vertical movement. Horizontal axis: no horizontal movement.
- **ACTION2 (DOWN)**: Move selected sprite down by 1px. Same constraints.
- **ACTION3 (LEFT)**: Move selected sprite left by 1px.
- **ACTION4 (RIGHT)**: Move selected sprite right by 1px.
- **ACTION5 (CYCLE)**: Cycle selection to next movable sprite in `xefwpvwoh` list (axes + shapes, excluding `pwbzvhvyzx`). Consumes a step.
- **ACTION6 (CLICK)**: `display_to_grid` conversion, then selects the clicked sprite. Prefers `gljpmsnsnx` shapes over axes. Toggles between vertical/horizontal axis when clicking same spot.
- **ACTION7 (UNDO)**: Pop last saved state from `usukvgwle` stack; restore positions.

Reflection computation (`dpdrrqfgvm`): For each shape, BFS bounces pixels off every axis (vertical: `new_x = 2*axis_x - x`, horizontal: `new_y = 2*axis_y - y`). Depth limit = 12 bounces. Reflected pixels rendered as `kddxpzgafq` color. Shapes tagged `xnpsicjqxc` only reflect off vertical axis; `fmxjsieygg` only off horizontal.

Auto-rotate: shapes tagged `dmfgetmeus` or `xnpsicjqxc` rotate 90 degrees when their associated axis moves past them (distance changes).

Collision: movement blocked if new position overlaps existing sprites (non-transparent pixel collision).

## Win Condition
Method `etzeptsuxx()`: Computes the full reflected grid via `jhajdrieqn()`. For EVERY target dot (`vrfjzqaker`), checks if `mqrjslqocj[y, x] >= 0` (i.e., some reflected or direct color covers that pixel). Returns True only when all target dots are covered.

On win: `mllqetvjb = True`, then `next_level()` on next step.

Loss: step counter reaches 0 (`zdrbnrjbr.tihzupejat()` returns False).

## Level Progression
9 levels. Grid is 8x8 for all. Each level provides:
- A `StepCounter` value for move budget
- Different arrangements of shapes, axes, and target dots
- Some levels have fixed shapes (`pwbzvhvyzx`), auto-rotating shapes, or single-axis-only reflectors

Complexity increases with more shapes, more target dots, and constrained reflection types.

## No-Source Discovery Strategy
1. Click on a colored shape to select it (orange cursor appears).
2. Use arrows to move it; observe reflected copies appearing symmetrically.
3. Click on the orange line(s) to select axes; move them to shift reflection plane.
4. Press ACTION5 to cycle through selectable objects.
5. Look for small teal dots (targets) -- these must be covered by reflections.
6. Use UNDO (ACTION7) to backtrack mistakes.
7-10. Map out which shapes reflect off which axes, test multi-bounce reflections.

## Key Patterns
- This is fundamentally a mirror/reflection geometry puzzle.
- Shapes create reflected images across orange axis lines with up to 12 bounce depth.
- Some shapes only reflect in one direction -- pay attention to special tags.
- Moving an axis can cause associated shapes to auto-rotate.
- The selection cursor (black L-shape) shows what is currently active.
- UNDO is critical: save-scum through trial positions.
- Target dots are small (1px), easy to miss -- they sit at layer -4 under everything.
- Step budget is tight on later levels; minimize wasted moves.
