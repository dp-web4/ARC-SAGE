# cd82 — Octagonal color-fill puzzle with directional basket and target pattern

## Toolset
Available actions: [ACTION1, ACTION2, ACTION3, ACTION4, ACTION5, ACTION6]. Directional movement (1-4), a "launch" action (5), and click (6). This is a spatial pattern-matching game: navigate a basket around an octagonal ring, select colors, and fill quadrants to match a target.

## Objects
- `eoqnvkspoa-pqwmeN-1` — Target pattern sprites (10x10). Each level has a different target with colored regions. These are the goal patterns to reproduce.
- `gkyfnkvrty` — L-shaped border/frame sprite (64x18) with color 4 (yellow) border and color 3 (green) fill. Provides the structural frame.
- `oaoosfneq-oaanwen` / `oaoosfneq-laopvne` — Basket sprites (horizontal and diagonal). The movable basket that deposits color into the central pattern. Diamond shape with arrow indicator.
- `xytrjjbyib` — Central 10x10 grid (the canvas being painted). Starts empty and gets filled with colors.
- `pqkenviek` — Color palette selectors (small squares at top). Clickable to change active color.
- `ydiwkzjkgl` — Selection indicator that marks the currently active color.
- `ctwspzkygu` — Launch button sprite (appears at horizontal positions only). Clickable shortcut to launch the basket.

## Interaction Model
- **Movement (ACTION1-4):** Moves the basket around an 8-position octagonal ring. The ring maps to a 3x3 grid (excluding center): positions 0-7 correspond to N, NE, E, SE, S, SW, W, NW. ACTION1=up row, ACTION2=down row, ACTION3=left col, ACTION4=right col. Center (1,1) is forbidden.
- **LAUNCH (ACTION5):** Animates the basket moving inward along its direction vector, painting a wedge of the central 10x10 canvas with the active color. The basket moves 7 steps in, paints the quadrant/octant via `rtjwayrycq()`, then returns. Different positions paint different regions:
  - Horizontal positions (0,2,4,6) paint rectangular halves (top/bottom/left/right 5 rows/cols).
  - Diagonal positions (1,3,5,7) paint triangular wedges.
- **CLICK (ACTION6):** Two targets:
  1. Color palette squares (`pqkenviek`): changes the active fill color.
  2. Launch buttons (`ctwspzkygu`): triggers the basket launch from that position.
- **Win check (`wvrremwltt`):** After each launch+return, compares the central canvas against the target pattern. The comparison uses a **mask that excludes the two diagonals** (pixels where i==j or i==9-j). Only the non-diagonal pixels must match.

## Win Condition
Method `wvrremwltt()`: Creates a 10x10 boolean mask where both main diagonals are False. Compares canvas pixels to target pixels at all True positions. **The canvas must match the target everywhere except on the two diagonals.** The diagonals are boundary lines between quadrants and are ignored.

## Level Progression
**6 levels.** All grids are 64x64.
- Level 1: 2 colors available (black, white). Simple half-fill pattern. No launch buttons.
- Level 2: 3 colors (black, white, pink). Diagonal wedge pattern introduced.
- Level 3-6: 7 colors available. Launch buttons (ctwspzkygu) added for quick access. Increasingly complex multi-color patterns with both horizontal and diagonal fills.

Action limit: 100 actions total. Counter displayed via `aunldyudeb` interface. Lose if exceeded.

## No-Source Discovery Strategy
1. **First action:** Press ACTION1/2/3/4. Watch a basket/arrow move around the edge of a circular arrangement.
2. **Observe:** Note the basket position changes. See a target pattern in the corner and an empty central square.
3. **Press ACTION5:** Watch the basket animate inward and paint a section of the central canvas.
4. **Hypothesis:** Navigate basket to correct position, select the right color, and launch to paint each section.
5. **Test color selection:** Click a color square at top. Basket changes color. Launch again and observe the painted region matches the new color.

## Key Patterns
- 8 positions form an octagonal ring. The `nicoqsvlg` table defines each: (type, x, y, rotation, dx, dy).
  - type: "horizontal" or "diagonal" determines paint shape.
  - rotation: visual orientation of basket sprite.
  - dx, dy: movement direction during launch animation.
- Position mapping (3x3 grid, center excluded):
  - 0=(0,1)=N, 1=(0,2)=NE, 2=(1,2)=E, 3=(2,2)=SE, 4=(2,1)=S, 5=(2,0)=SW, 6=(1,0)=W, 7=(0,0)=NW.
- Horizontal fills paint 5 rows or columns. Diagonal fills paint triangular regions using loop `hzmesgtof[i, 0:i+1]` etc.
- Active color stored in `knqmgavuh` (default 15 = white).
- Launch animation: 7 steps out, pause, 7 steps back. Total ~14 frames per launch.
- Background=5 (grey), padding=4 (yellow).
- The basket and launch button sprites share color with the active fill color (remapped on color change).
