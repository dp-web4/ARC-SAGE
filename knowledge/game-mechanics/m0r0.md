# m0r0 — Mirrored Maze: Navigate Paired Tokens Through a Maze with Symmetric Movement

## Toolset
Available actions: [ACTION1=UP, ACTION2=DOWN, ACTION3=LEFT, ACTION4=RIGHT, ACTION5=SELECT, ACTION6=CLICK].
Arrows move tokens. Click selects/deselects individual tokens. Two modes: "group move" (arrows move all paired tokens with mirrored directions) and "single move" (click a token, then arrow to move just that one).

## Objects
- **`qzfkx-ubwff-idtiq`** and **`qzfkx-ubwff-crkfz`**: Two movable paired tokens (1x1, color 10/green). Tagged `sys_click` and `pxwnx`. These move symmetrically in group mode. "ubwff" pair.
- **`qzfkx-kncqr-idtiq`** and **`qzfkx-kncqr-crkfz`**: Second pair of tokens (1x1, color 10). Only in later levels. "kncqr" pair. Move with inverted Y compared to ubwff pair.
- **`jggua-LevelN`**: Maze walls (color 0/black). Various shapes per level. Tagged `jggua`. Block movement.
- **`cvcer`**: Obstacle tokens (1x1, color 9/maroon). Tagged `sys_click` and `nhiae`. Can be individually selected and moved. Block paired token movement.
- **`wyiex`**: Trap/hazard tiles (1x1, color 8/grey). Tagged `wyiex`. Landing on these triggers a flash animation and resets to previous positions.
- **`dfnuk-COLOR`**: Door/barrier sprites (3x1 vertical, various colors 12/14/15). Block movement when active. Not clickable.
- **`hnutp-COLOR`**: Switch/trigger tiles (1x1, matching colors). When a paired token sits on a switch, the matching door is removed (REMOVED interaction).
- **Step counter UI**: Two progress bars (top and bottom rows), color 5, 150 steps max.
- **Background pattern UI** (`kyrgqgqaes`): Renders a checkerboard/split pattern using `npwxa` colors from level data. Shows cvcer borders and wyiex crosshatch patterns.

## Interaction Model
- **CLICK (ACTION6)**: If clicking a `cvcer` obstacle, selects it for individual movement (turns color 11/cyan, paired tokens turn color 1/blue). If clicking empty space, deselects and returns to group mode (paired tokens turn color 10/green).
- **Arrows in GROUP MODE** (`vmcbq=True`): All active (non-merged) paired tokens move simultaneously with mirrored directions:
  - `ubwff-idtiq`: moves (dx, dy) as given
  - `ubwff-crkfz`: moves (-dx, dy) -- X is mirrored
  - `kncqr-idtiq`: moves (dx, -dy) -- Y is mirrored
  - `kncqr-crkfz`: moves (-dx, -dy) -- both mirrored
- **Arrows in SINGLE MODE** (`vmcbq=False`): Moves only the selected `cvcer` sprite.
- **Collision**: Tokens cannot move through maze walls (`jggua`), other `cvcer` obstacles, or active door barriers (`dfnuk`).
- **Trap tiles**: If a paired token lands on a `wyiex` tile after group movement, a flash animation plays (6 frames alternating color 11/10) and all tokens snap back to their pre-move positions.
- **Merging**: After group movement, if two paired tokens that were adjacent before the move end up swapping/overlapping, they merge to the midpoint. If two tokens share the same position, they are both marked as merged (INTANGIBLE) and removed from active play.

## Win Condition
After each group move, count remaining active paired tokens (not in `fqunj` merged set, not INTANGIBLE). **Win = 0 active paired tokens remaining.** All pairs must be merged by moving them to the same position.

## Level Progression
**6 levels**. Grid sizes: 11x11, 13x13, 13x13, 11x11, 15x15, 13x13. Changes:
- **Level 1**: 2 paired tokens (ubwff pair), simple ring maze, no obstacles.
- **Level 2**: 2 paired tokens, cross-shaped maze, many `wyiex` traps.
- **Level 3**: 2 paired tokens + 3 `cvcer` obstacles, more complex maze.
- **Level 4**: 2 paired tokens + 1 `cvcer`, maze with traps, ring layout with center barriers.
- **Level 5**: 2 paired tokens, colored doors (`dfnuk`) and switches (`hnutp`). Standing on a switch of matching color opens the corresponding door.
- **Level 6**: 2 paired tokens + 1 `cvcer`, doors/switches, walled perimeter, complex multi-room layout.

Key data per level: `npwxa` = [color1, color2] for background pattern.

## No-Source Discovery Strategy
1. **First action**: Press an arrow key (e.g., RIGHT). Observe two tokens move -- one right, one left (mirrored X).
2. **Observe**: Note the maze walls block movement. One token may move while the other is blocked.
3. **Hypothesis**: The two tokens must be brought to the same position to merge. Movement is mirrored.
4. **Test**: Try to navigate both tokens toward each other. When they overlap, they should merge and disappear.
5. **When ready**: Once you understand the mirroring, plan paths that bring pairs together. Note that clicking a small obstacle lets you move it out of the way individually.

## Key Patterns
- Movement mirroring: X-axis for ubwff pair, Y-axis for kncqr pair, both for crkfz variants.
- 150-step hard limit.
- Traps (`wyiex`) reset positions on contact -- avoid them or plan around them.
- Doors open when a paired token sits on the matching-color switch. The door check happens after every move via `dssyxgdgjg()`.
- Merging requires two tokens to occupy the same cell after movement. Adjacent tokens that swap positions also merge (midpoint merge logic).
- `cvcer` obstacles block paired token movement but can be relocated by clicking and arrow-moving them individually.
