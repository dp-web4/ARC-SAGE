# r11l -- Pick-and-place creature assembly with animated movement

## Toolset
Available actions: [ACTION6 (CLICK only)]. Click-only interaction.
Implies: spatial puzzle where you click to select limbs and click destinations to place them, assembling creatures at target locations.

## Objects
| Naming Convention | Role | Description |
|-----|------|-------------|
| `bdkaz-{color}` | Body (creature head) | The main body sprite for each color group. Position is auto-calculated as centroid of its limbs. |
| `bdkazLeg-{color}` | Limb (leg) | Movable limbs tagged `sys_click`. These are the pieces you select and move. Highlighted in color 0 (black) when selected, color 3 (green) otherwise. |
| `kzeze-{color}` | Target | Goal position where the assembled creature should end up. Win when body overlaps target AND colors match. |
| `bvzgd-Level{N}` | Obstacle (wall) | Large background sprites that block movement. Legs cannot be placed on walls. |
| `qtwnv-Level{N}` | Danger zone | Collision with these triggers a penalty counter. 5 penalties = lose. |
| `xigcb-{variant}` | Stamp/overlay | Stamps that get absorbed into `yukft`-type bodies on collision, adding color patterns. |
| `mpjrv-bdkaz` / `mpjrv-kzeze` | Flash indicator | Blinking sprites for feedback when body reaches target or danger zone. |

Custom Camera: standard Camera with two interfaces -- step counter bar (left edge, vertical) and line-drawing overlay (connects bodies to their limbs with lines).

## Interaction Model
- **CLICK (on a limb)**: Selects that limb. The previously selected limb unhighlights (color 0 -> 3), the new one highlights (color 3 -> 0). Selection order cycles through limbs sorted by distance from origin.
- **CLICK (on empty space)**: Moves the currently selected limb to the clicked position (centered). This triggers animation:
  1. Limb interpolates from start to destination over `gfwuu` frames (default 1, so instant).
  2. Body position recalculates as centroid of all its limbs (`bdahxidrjf`).
  3. If body collides with a danger zone (`qtwnv`), penalty counter increments. After 5 penalties, you lose. The body flashes for ~20 frames then the limb snaps back.
  4. If body collides with a wall (`bvzgd`), the click is rejected entirely (no movement).
  5. If body has `yukft` name prefix and collides with stamp sprites (`xigcb`), the stamp's non-transparent pixels are merged into the body sprite.
  6. After movement completes, checks if ALL bodies overlap their targets AND have matching color sets. If yes, triggers win animation (blinking at target locations).
- **Animation blocks input**: While `bmtib` is True, each `step()` call advances the animation by one frame. Actions during animation are consumed.
- **Step limit**: 60 total actions. Lose when exceeded.

## Win Condition
Method embedded in `tpjhojnaoa()` (line ~1744-1768): For each color group, the body (`bdkaz`) must collide with its target (`kzeze`). For groups without a direct body (stamp-assembled bodies), checks if any `yukft`-type body collides with the target AND has matching color set (`trfdmyhcav` -- unique non-zero pixel colors must match). Colors named `anqcf` are excluded from the check.

## Level Progression
6 levels. All 64x64 grid.
- **Level 1**: 1 creature (1 body, 2 limbs, 1 target). Simple placement.
- **Level 2**: 2 creatures (2 colors), introduces walls with mirror flip, adds danger zones.
- **Level 3**: 2 creatures, more limbs (3-4), more complex wall layout, danger zones.
- **Level 4**: 3 creatures, introduces `anqcf` (neutral target excluded from win check), stamp overlays (`txxvz-anqcf`).
- **Level 5**: 2 `yukft`-type creatures (stamp assembly), 4 stamp sprites (`xigcb`). Must absorb correct stamps to match target colors.
- **Level 6**: 2 `yukft` creatures, 9 stamp sprites, most complex color matching.

Key data structure per color group (`brdck` dict):
- `kignw`: body sprite (or None for stamp-assembled)
- `mdpcc`: list of limb sprites
- `xwdrv`: target sprite (or None)

## No-Source Discovery Strategy
1. **First click**: Click on a small colored square (a limb). It should highlight/change color.
2. **Observe**: Note which piece changed. Look for a larger body sprite nearby connected by lines.
3. **Hypothesis**: "Clicking selects a limb. Clicking elsewhere moves it. The body follows as the centroid of limbs."
4. **Test**: Click on empty space away from walls. Watch the limb move and body reposition. Check if step counter decreases.
5. **Ready to solve**: Identify all target positions (different colored squares), figure out which limbs belong to which body, and move limbs so each body lands on its matching target.

## Key Patterns
- Bodies auto-position at centroid of their limbs -- you never move bodies directly. The centroid calculation (`bdahxidrjf`): average the center points (x + width/2, y + height/2) of all limbs, then position body so its center is at that average.
- Limbs are sorted by distance from origin (`(x^2 + y^2)^0.5`) for selection cycling. The initial selection is the closest limb to origin.
- Limb highlighting: selected = color 0 (black), unselected = color 3 (green). The `pprcmbqpwi` method handles swapping.
- Wall collision = rejected move (free, no step cost). Checked by temporarily moving the limb, testing collision with `bvzgd` sprites, then reverting.
- Danger zone collision = penalty counter +1 + animated revert (costs a step). After 5 penalties across the level, you lose. The penalty animation:
  - Body flashes for ~20 frames (blink sprite added/removed every 2 frames)
  - After flash completes, `cjcex=True` triggers reverse animation (limb slides back to start)
  - After reverse completes, action is consumed
- Stamp absorption (`iciufrewti`): only for `yukft`-named bodies. When body collides with a stamp (`xigcb`), all non-transparent, non-zero pixels from the stamp are copied into the body sprite. The stamp is removed permanently.
- Color matching for win (`trfdmyhcav`): `{int(c) for c in np.unique(body.pixels) if c > 0}` must equal same set for target. Both must have identical sets of positive pixel colors.
- Groups named `anqcf` are excluded from the win check (line 1748: `if "anqcf" in hzottalifh: continue`).
- 60-action hard limit across all levels. Step counter is vertical bar on left edge of display.
- Animation timing: `gfwuu=1` means interpolation completes in 1 frame (instant movement). But the overall animation lifecycle (move -> check collisions -> flash -> revert) spans many frames.
- Line overlay (`lbnpnbcxsr`): renders Bresenham lines from each limb center to body center in display coordinates. Uses `camera._calculate_scale_and_offset()` for coordinate transforms. Line color = 1 (dark blue), only drawn over background (color 5 or 10).
- Limb click detection (line 1812): checks if grid coordinate falls within limb's bounding box.
- Destination centering: clicked position is adjusted by `(-width/2, -height/2)` so the limb is centered on the click point.
- Win animation: all target positions flash with `mpjrv-kzeze` sprites for ~20 frames, then `next_level()` is called.
- Background/padding both color 5 -- display is seamless grey.
