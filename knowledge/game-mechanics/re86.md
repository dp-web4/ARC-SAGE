# re86 -- Shape-matching puzzle with color zones, deformation, and pattern matching

## Toolset
Available actions: [ACTION1 (UP), ACTION2 (DOWN), ACTION3 (LEFT), ACTION4 (RIGHT), ACTION5 (CYCLE SELECTION)].
Five actions. Four directional plus a selection cycler. No click action. This implies a puzzle where you select among multiple movable objects and slide them into position. Movement step size: `ilmaurgzng` = 3 pixels.

## Objects
| Tag | Role | Description |
|-----|------|-------------|

## Interaction Model
**Selection system:** One piece is "active" at a time, indicated by center pixel = 0 (black). ACTION5 cycles: finds the current active piece among all `vfaeucgcyr` sprites (the one where `wfftxovxaa` returns True, i.e., center == 0), advances to the next one in list order (wrapping around). The previously active piece's center is restored via `tgawbgxjra()`:
- `nogegkgqgd` pieces: center = -1 (hollow)
- `ldkpywfara` pieces: center = body color (filled)
- Default pieces: center = body color if piece is cross-shaped (center row/col are the only solid ones), else -1 (hollow)
The newly active piece gets center = 0 (black).

**Movement (ACTION1-4):** Moves the active piece by 3 pixels via `ufgmbvuqnv()`. Validation:

1. **Bounds check:** Piece center must stay within camera bounds (0 to 64 on both axes).

2. **Wall collision (`miqpqafylc`):** If the piece collides with a wall after moving:
   - **`nogegkgqgd` (deformable) pieces:** The piece reshapes. Horizontal movement + width > 6: becomes taller/narrower (width-3, height+3). Vertical movement + height > 6: becomes wider/shorter (height-3, width+3). If too small, movement blocked. Piece snaps to nearest multiple-of-3 position. If horizontal and moving left, movement is also reverted.
   - **Normal pieces:** Border-shifting logic. If the piece's leading edge overlaps the wall along one axis, a border column/row is removed from the collision side and added to the opposite side, effectively "squeezing" the piece. If both axes overlap, movement is blocked entirely.

3. **Paint zone collision (`ozhohpbjxz`):** After movement and wall handling, if the piece overlaps a paint zone and the piece's color differs from the zone's color (checked via `hfoimipuna`), recoloring begins. Overlapping pixels are immediately set to the zone's color. The function returns the zone sprite, triggering the animation state.

**Color-spreading animation:** When `zrermyobpw` (active zone) and `gwwpnjmvvo` (recoloring piece) are set, `step()` runs the flood-fill. Each frame: finds all pixels matching the zone color, then for each, recolors all adjacent non-transparent pixels. When the entire piece matches the zone color, the center is set to 0 (black/active) and the animation ends. No player input is processed during this animation.

**Center pixel semantics after movement:** After every move, center pixel is recalculated per piece type (hollow, filled, or conditional), then overwritten to 0 (black) to mark it as active.

## Win Condition
Method `cdjxpfqest()` (lines 1894-1918):
1. Copy the `pseflysmdl` canvas (64x64 all -1).
2. Composite all `vfaeucgcyr` (movable pieces) onto the canvas, overwriting non-(-1) pixels.
3. For each `vzuwsebntu` (target outline): check that every pixel that is not -1 and not color 4 matches the corresponding canvas pixel.
4. If ALL targets match, return True -> `self.next_level()`.

Key detail: color 4 in target pixels is "don't care" (wildcard). Pieces must be positioned, shaped, AND colored correctly to match targets. Overlapping pieces overwrite each other on the canvas -- be careful about layering.

If step counter reaches 0, `self.lose()`.

## Level Progression
8 levels. All use 64x64 grid. Step budgets: 100, 100, 200, 200, 250, 200, 300, 400.

- **Level 1**: 1 movable piece, 1 target. Simple positioning tutorial. 100 steps.
- **Level 2**: 2 movable pieces, 1 target. Position both correctly. 100 steps.
- **Level 3**: 2 movable pieces, more complex shapes. 200 steps.
- **Level 4**: Many pieces (5+), multiple targets and color zones. Introduces spatial complexity and recoloring. 200 steps.
- **Level 5**: Multiple pieces with initial rotations (.set_rotation(90/180)). Pieces start rotated. 250 steps.
- **Level 6**: Deformable pieces (`nogegkgqgd`) and paint zones central to solution. Must use wall collisions to reshape pieces. 200 steps.
- **Level 7**: Complex layout with many rotated pieces, walls, and paint zones. 300 steps.
- **Level 8**: Maximum complexity. Many pieces at varied positions around canvas edges, walls, paint zones, rotations. 400 steps.

What changes: number of pieces, presence of walls, paint zones, piece sub-tags (nogegkgqgd/ldkpywfara), initial rotations, and spatial layout complexity.

## No-Source Discovery Strategy
1. Press a direction. Observe one rectangle moves. Note the one with a black center dot -- that is the active piece (1 action).
2. Press ACTION5 (cycle). Observe the black dot moves to a different piece. Previous piece's center changes (1 action).
3. Move the newly selected piece. Confirm independent movement. Note 3-pixel step size (1-2 actions).
4. Move a piece toward a wall. Observe deformation (shape change) or border-shift (1-2 actions).
5. Move a piece into a colored zone. Observe flood-fill recoloring animation. Note input is paused during animation (1-2 actions).
6. Compare target outlines (static colored shapes) to movable pieces. Targets show required final arrangement (observation).
7. Position a piece exactly on a target. If all targets are covered correctly, level completes (1-2 actions).

## Key Patterns
- **Selection cycling is list-order.** ACTION5 always advances to the next piece in sprite list order, wrapping from last to first.
- **Black center dot (color 0) = active piece.** Always visible. Other pieces show natural center (filled or hollow depending on type).
- **Movement in steps of 3 pixels.** Plan positions in multiples of 3 for alignment.
- **Deformable pieces (`nogegkgqgd`) are key to later levels.** Intentional wall collisions change aspect ratio. Width-3/height+3 or height-3/width+3. Required to match target dimensions.
- **Paint zones recolor permanently.** Once a piece enters a zone of different color, it fully recolors after the animation. Plan color changes before final positioning.
- **Color 4 in targets is wildcard.** Those pixels are not checked. Allows partial coverage.
- **Canvas compositing matters.** Win check renders all pieces onto a shared canvas. Overlapping pieces overwrite each other. Ensure no piece obscures another's required target area.
- **Rotated pieces stay rotated.** Initial `.set_rotation()` changes pixel layout. A 3-wide-by-9-tall piece rotated 90 degrees becomes 9-wide-by-3-tall.
- **Normal piece border-shifting is directional.** When a non-deformable piece hits a wall, the border on the collision side retracts and appears on the opposite side. This changes the piece's cross shape, potentially required for matching targets.
- **Flood-fill animation blocks input.** During recoloring, no movement is accepted. Each frame spreads the new color one ring outward from existing recolored pixels.
- **Step budgets scale with complexity.** Level 1: 100, Level 8: 400. Later levels give more room but deformation/recoloring mistakes can be costly to fix.
