# ls20 -- Maze navigation with shape/color/rotation matching and fog of war

## Toolset
Available actions: [ACTION1 (UP), ACTION2 (DOWN), ACTION3 (LEFT), ACTION4 (RIGHT)]
This is a grid-based maze navigation game. The player moves a cursor through a walled maze, collecting pickups and matching a carried shape+color+rotation to goal slots. Camera is 16x16 with custom rendering.

## Interaction Model
- **ACTION1 (UP)**: Move player cursor up by `height` pixels (one grid cell). Checks destination for:
  - Wall (`ihdgageizm`): blocked, position reverts
  - Goal (`rjlbuycveu`): if current shape/color/rotation matches, the goal is collected (marked as done). If not matched, triggers error flash (5 frames).
  - Step refill (`npxgalaybz`): resets step counter, pickup is removed
  - Shape modifier (`ttfwljgohq`): cycles `fwckfzsyc` through shape list
  - Color modifier (`soyhouuebz`): cycles `hiaauhahz` through color list
  - Rotation modifier (`rhsxkxzdjz`): cycles `cklxociuu` through rotation list
  
- **ACTION2 (DOWN)**, **ACTION3 (LEFT)**, **ACTION4 (RIGHT)**: Same mechanics in respective directions.

- **Step counter**: Decrements by `StepsDecrement` (1 or 2) per move. When depleted, the player gets a "retry" -- position resets to start, step counter refills, lives decrease by 1. After 3 retries (0 lives), you lose.

- **Patrolling enemies** (`gbvqrjtaqo`): `twkzhcfelv` objects that move along predefined paths. After each player step (if step counter is valid), enemies check if they overlap with the player cursor. If so, they trigger an animation sequence.

- **Modifier pickups in containers**: `dboxixicic` objects track which modifiers are inside which container rooms. Each step, containers cycle their modifiers.

- **Match checking** (`bejndxqqzf`): Returns True when the player's current shape index, color index, and rotation index all match the goal's expected values. When matching during L1, a highlight (`hoswmpiqkw`) appears on the relevant goal.

- **Fog of war**: When `Fog = True` (Level 7), rendering masks everything beyond radius 20 from the player position. The carried shape is rendered separately in a fixed position.

## Level Progression
7 levels. Step counter is 42 for all levels, with StepsDecrement of 1 or 2.
- L1: 1 goal, simple maze, no fog. Shape=5, Color=9, Rotation=270 start; Goal needs Color=9, Rotation=0.
- L2: 1 goal, more complex maze.
- L3: 1 goal, different start/goal color mismatch.
- L4: 1 goal, different start shape.
- L5: 1 goal, different start shape+color.
- L6: 2 goals with different targets, more walls and enemies.
- L7: 1 goal, FOG OF WAR enabled. Must navigate blind beyond radius 20.

## No-Source Discovery Strategy
1. Press directional keys to observe movement -- note grid-cell-sized jumps (1-2 actions)
2. Notice the shape display in corner -- note its color and rotation
3. Move onto a colored pickup tile -- observe shape/color/rotation change (1-3 actions)
4. Locate goal slots (bordered squares with target shapes inside)
5. Move onto a goal with wrong config -- observe error flash
6. Adjust shape/color/rotation by hitting modifier pickups, then return to goal
7. Note the step counter bar and lives display at bottom
8. If fog appears (L7), note the visibility radius mechanic

## Key Patterns
- Shape, color, and rotation must ALL match the goal simultaneously.
- Modifier pickups cycle through options -- plan your route to hit the right sequence.
- Step counter is tight (42 steps with decrement 1-2). Retries give 3 lives but reset position.
- Goals are removed when matched, so order matters if multiple goals exist.
- Enemies patrol and can interfere -- avoid their paths.
- Fog of war in L7 requires memorizing the maze layout from earlier exploration.
- Container rooms cycle their modifier pickups each step -- timing matters.
- When a match is possible (shape/color/rotation matches any uncollected goal), a highlight indicator appears. Use this as feedback.
