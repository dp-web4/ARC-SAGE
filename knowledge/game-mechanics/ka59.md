# ka59 -- Sokoban-style push puzzle with explosions and step counter

## Toolset
Available actions: [ACTION1 (UP), ACTION2 (DOWN), ACTION3 (LEFT), ACTION4 (RIGHT), ACTION6 (CLICK)]
This is a grid-based push puzzle. Movement step size is 3 pixels (`jesnwclftg = 3`). CLICK selects which box-sprite to control. Step counter limits total moves.

## Interaction Model
- **ACTION1-4 (UP/DOWN/LEFT/RIGHT)**: Move the selected box by 3 pixels in the given direction. Decrements step counter. On movement:
  1. Attempt to move selected box (`ascpmvdpwj`).
  2. If collision with walls (`divgcilurm`) or gulches (`vwjqkxkyxm`) -- movement blocked, position reverted.
  3. If collision with another pushable object (`xlfuqjygey`/`nnckfubbhi`/`gobzaprasa`) -- those objects are pushed. Push is recursive: if the pushed object also collides, it tries to push further. If chain is blocked by wall, entire move reverts.
  4. After movement, enemies chase toward `nnckfubbhi` by up to 6 steps of 1 pixel each. If enemy reaches target, you lose.
  5. Bombs (`gobzaprasa`) are checked: rows are consumed (set to color 12). When fully consumed, bomb explodes.
  
- **ACTION6 (CLICK)**: Uses `display_to_grid` to convert click coordinates to game coordinates. Finds an `xlfuqjygey` sprite at that position and selects it as the active box (changes its interior color to 0/active).

- **Explosion system**: When a bomb fully consumes, it enters the `zglbibxjth` explosion list. Over 3 animation frames, explosion sprites expand outward based on the bomb's rotation (0/90/180/270 determines blast direction). Explosion pushes nearby objects in the blast direction. After frame 3, the bomb is blanked out and pushed objects settle.

- **Box borders**: On `obrnalfvlv()`, each side of the selected box's border is drawn (color 14) unless an adjacent object is touching that side. Adjacent objects cause that border side to be cleared (color 0), visually showing connectivity.

## Win Condition
Method `cbdaoltbck()`:
```python
# All rktpmjcpkt (goals) must be covered by xlfuqjygey (boxes)
for aewfaavdxp in rktpmjcpkt_sprites:
    if not vbolnamxei(aewfaavdxp, xlfuqjygey_sprites):
        return False
# All ucjzrlvfkb (secondary goals) must be covered by nnckfubbhi
for rykimoibzb in ucjzrlvfkb_sprites:
    if not vbolnamxei(rykimoibzb, nnckfubbhi_sprites):
        return False
return True
```
`vbolnamxei` checks that a box is positioned exactly inside the goal marker: `box.x == goal.x + 1, box.y == goal.y + 1, box.height == goal.height - 2, box.width == goal.width - 2`.

Win is checked after each move. If won, advances to next level. If step counter reaches 0, you lose.

## No-Source Discovery Strategy
1. CLICK on different colored rectangles to see which one becomes "selected" (interior color changes) -- 2 actions
2. Press directional keys to move the selected box -- 2 actions
3. Push the box into another object to observe push mechanics -- 1 action
4. Notice the step counter bar shrinking at bottom row -- passive observation
5. Look for goal markers (slightly larger rectangles) and try to push boxes into them
6. If bombs exist, push them and observe the explosion animation
7. If enemies exist, observe their chase behavior after each move
8. Click to switch between boxes if multiple exist
9. Count goals and boxes to understand the level objective

## Key Patterns
- This is a Sokoban variant: push boxes onto goal markers.
- CLICK to select which box you control. Only one box is active at a time.
- Push is recursive -- you can push chains of objects, but any wall stops the whole chain.
- Bombs explode directionally based on their rotation. Use explosions strategically to push objects.
- Enemies move 6 steps toward `nnckfubbhi` after each player move -- plan enemy pathing.
- Border rendering shows which sides of the active box are adjacent to other objects.
- Step counter is per-level. Running out loses the level.
- Goal alignment must be pixel-perfect: box must be exactly 1px inset from goal on all sides.
