# lp85 — Tile-rotation puzzle with numbered cycles on grid tracks

## Toolset
Available actions: [ACTION6 (click only)]. Pure click game with left/right arrow buttons flanking each track. This is a cyclic permutation puzzle: rotate tiles along predefined tracks to place them at goal positions.

## Objects
- `button_N_L` / `button_N_R` (tags like `button_A_L`, `button_13_R`, etc., plus `sys_click`) — Arrow buttons. Left (L) and right (R) buttons for each named track (A, B, C, or numbered 1-27+). Color 8 (azure) for L, color 14 (grey) for R. Clicking rotates all tiles on that track one position.
- `tile` (tag) — Colored tile sprites (2x2). These are the movable pieces. Various colors: 9 (maroon), others.
- `bghvgbtwcb` (tag) — Goal marker sprites with corner dots (4x4, color 11 at corners, transparent center). Tiles must be placed at these positions.
- `fdgmtkfrxl` (tag) — Secondary goal markers (for "goal-o" matching).
- `goal` (tag) — Goal target sprites that sit at goal positions for validation.
- `goal-o` (tag) — Secondary goal target sprites.
- Various colored tiles: `ekpijbeoel`, `kisflvfvsy`, `slefrnjymo`, `tddzizwhzf`, `odkpvwbihk`, `aydooxtcli` — Different colored 2x2 or 3x3 tile sprites.

## Interaction Model
- **CLICK (ACTION6):** Uses `display_to_grid` for coordinate mapping. Clicks target button sprites. The button tag encodes: `button_{trackID}_{direction}`.
  - `trackID`: identifies which track/cycle (e.g., "A", "2", "13").
  - `direction`: "L" (left/backward) or "R" (right/forward).
  - On click, calls `chmfaflqhy()` which computes a cyclic permutation: every tile on the track shifts to the next position in the cycle. "R" shifts forward (each tile at position N moves to position N+1, last wraps to 1). "L" shifts backward (N moves to N-1, first wraps to last).
- **Track data (`izutyjcpih`):** A nested dictionary keyed by level name, then track name. Each track is a 2D grid of integers where numbers define cycle order. -1 = empty. Tiles at numbered positions rotate along the cycle.
- **Position scaling:** Tile grid positions are multiplied by `crxpafuiwp = 3` to get pixel coordinates.

## Win Condition
Method `khartslnwa()`: For each `bghvgbtwcb` goal marker, check if there's a sprite with tag `goal` at position (marker.x+1, marker.y+1). For each `fdgmtkfrxl` goal marker, check if there's a sprite with tag `goal-o` at (marker.x+1, marker.y+1). **All goal positions must have the correct tile type sitting on them.** Returns True only when every goal marker is satisfied.

## Level Progression
**8 levels.** Grid sizes vary: 32x19 (L1), 41x41 (L2), larger for later levels.
- Level 1: 1 track ("A"), 20 tiles in a rectangular loop. StepCounter=13.
- Level 2: 3 tracks ("A", "B", "C"), two nested loops plus a line. StepCounter=60.
- Level 3+: More tracks, more complex interlocking cycles, larger grids. Multiple goal markers.
- Key data per level: `StepCounter` (energy/move limit), `level_name` (maps to track definitions in `izutyjcpih`).

Track shapes from the data: rectangular loops (tiles cycle around a rectangle), linear rows (tiles shift left/right), and complex interlocking paths.

## No-Source Discovery Strategy
1. **First click:** Click an arrow button (L or R shaped, at edges of the grid). Watch tiles shift positions along a path.
2. **Observe:** Tiles move in a cycle -- one direction shifts them clockwise, the other counterclockwise. Notice corner-dot markers at certain positions.
3. **Hypothesis:** Rotate tiles along tracks until the right colors land on the goal positions.
4. **Test:** Click the opposite arrow. Tiles shift the other direction. Find which tracks share tiles or intersect.
5. **Strategy:** With multiple tracks, identify which track controls which goal position. Short cycles need fewer moves; plan to minimize total clicks within the step limit.

## Key Patterns
- Position scale factor: `crxpafuiwp = 3`. Grid coordinates to pixel = multiply by 3.
- Step counter is the move limit. Each valid button click costs 1 step. Lose when steps reach 0.
- Energy bar rendered along left edge (column 0, all 64 rows). Color 14 = remaining, color 5 = spent.
- Level progress indicators at top: small 4px bars at y=1, spaced 5px apart. Shows completed vs remaining levels.
- Track cycle lengths vary: 20 tiles (L1), 26 tiles (L2 track A), 10 tiles (L2 track B), etc.
- Duplicate position filtering: if two clickable sprites share (x,y), only the first is kept clickable.
- Background=4 (yellow), padding=3 (green).
- Button shapes: L buttons are 3x4 with left-pointing arrow shape, R buttons are 3x4 with right-pointing arrow shape.
